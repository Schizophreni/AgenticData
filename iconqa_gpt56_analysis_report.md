# IconQA 数据集深入分析报告

## 一、总体概述与综合评价

IconQA 是一个面向小学数学视觉问答的多任务数据集，核心目标不是识别自然图像中的物体，而是理解由抽象、彩色、扁平化图标构成的视觉场景，并结合问题文本完成计数、几何、空间关系、概率、模式等推理。

根据本地全量统计，数据集共包含 **107,439 道题目**，与论文规模一致。三个子任务的实际分布为：

| 子任务 | 数量 | 占比 |
|---|---:|---:|
| choose_img | 57,672 | 53.7% |
| choose_txt | 31,578 | 29.4% |
| fill_in_blank | 18,189 | 16.9% |
| 总计 | 107,439 | 100% |

总体而言，IconQA 的优点在于：

1. **规模足够大**，适合训练视觉语言模型或进行系统性基准测试；
2. **任务形式多样**，同时覆盖图像选择、文本选择和开放式填空；
3. **问题与小学数学知识体系对齐**，具有明确的年级、技能和课程标准背景；
4. **图像具有特殊性**，模型不能完全依赖自然图像预训练中形成的常规物体识别能力；
5. **问题平均较短，但视觉-语言组合关系明确**，适合研究视觉感知、符号推理和多步组合能力。

但它并不是一个完全均衡、完全适合作为通用 VLM 能力评测的数据集。其主要问题包括：

- 题型分布明显偏向 choose_img；
- 答案高度集中于小整数；
- 多数选择题只有两个选项；
- 选择题答案位置分布受到选项数量影响；
- 年级、技能、题型和图像模板之间可能存在强相关；
- 很多问题可能通过浅层视觉计数、文本先验或模板记忆解决；
- 对当前大型多模态模型而言，部分题目的语言难度较低，但抽象视觉推理仍具有一定挑战。

因此，IconQA 更适合作为 **小学数学视觉推理数据集、抽象图标视觉理解数据集和受控场景多模态训练集**，而不应直接被解释为全面衡量通用视觉智能的基准。

# 二、数据规模、结构与任务设计

## 2.1 规模与划分

数据集实际 split 统计为：

| Split | 数量 | 占比 |
|---|---:|---:|
| train | 64,462 | 60.00% |
| val | 21,488 | 19.999% |
| test | 21,489 | 20.00% |

论文或 metadata 中通常记录为：

- train：64,462
- val：21,489
- test：21,489

本地统计显示 **val 少 1 条**，但三个 split 的总数仍为 107,439。这属于轻微元数据不一致，不影响主体实验，但在复现实验、自动构建数据加载器或进行严格审计时需要记录和修正。

数据文件结构较清晰：

```text
iconqa/
  train/
    choose_img/
    choose_txt/
    fill_in_blank/
  val/
  test/
```

每道题通常包含：

- `data.json`
- `image.png`
- 可选的 `choice_*.png`
- `question`
- `hint`
- `choices`
- `answer`
- `ques_type`
- `grade`
- `label`
- `split`

实际统计显示：

- 主图像：107,439 张，基本对应每道题一张主图；
- 选项图：150,594 张，主要来自 choose_img；
- 因而 choose_img 的图像输入不是简单的“问题图像 + 文本选项”，而是“问题场景图 + 多个候选视觉答案”。

这一点使 choose_img 更接近视觉匹配、视觉分类和视觉推理的结合任务。

---

## 2.2 三种任务的功能差异

### 1. fill_in_the_blank

数量为 18,189，占 16.9%。

示例：

> How many shapes are purple?  
> 答案：4

该任务要求模型输出短文本，通常是数字或简短词语。它的优点是：

- 不受候选选项限制；
- 更接近开放式视觉问答；
- 能够测试模型是否真的生成正确答案；
- 更容易暴露“选项排除”以外的视觉推理问题。

但它也存在明显风险：答案词汇非常小，且大量答案是小整数，因此模型可能学习答案先验，而不完全依赖图像。

---

### 2. choose_img

数量为 57,672，占 53.7%，是数据集中占比最高的子任务。

示例：

> Which shape is a cylinder?

输入通常包括主图像和多个候选图像，模型需要选择正确候选。

该任务比较适合测试：

- 视觉属性识别；
- 形状和物体类别匹配；
- 图像之间的细粒度比较；
- 视觉语言指代表达；
- 选项级视觉推理。

其不足是，选择机制容易降低输出难度。即便模型不能完整生成答案，只要能够比较候选图像，就可能取得较高准确率。尤其在只有两个选项时，随机准确率已经达到 50%。

---

### 3. choose_txt

数量为 31,578，占 29.4%。

示例：

> If you select a marble without looking, how likely is it that you will pick a black one?

选项：

- impossible
- probable
- unlikely
- certain

该任务通常更接近传统多项选择视觉问答，考查：

- 视觉场景理解；
- 数量和比例判断；
- 数学概念到自然语言答案的映射；
- 概率、比较、时间、测量等概念理解。

choose_txt 的优势是选项语义更丰富，较适合测试答案概念选择；但如果正确答案和某些选项存在明显语言或频率偏差，也可能出现非视觉 shortcut。

# 三、年级、技能和问题复杂度分析

## 3.1 年级分布

年级统计为：

| 年级 | 数量 | 占比 |
|---|---:|---:|
| kindergarten | 37,500 | 34.9% |
| prek | 18,049 | 16.8% |
| grade1 | 26,944 | 25.1% |
| grade2 | 24,946 | 23.2% |
| grade3 | 未出现在本地汇总中 | — |

根据给出的实际数据，样本主要集中在：

- kindergarten：34.9%
- prek：16.8%
- grade1：25.1%
- grade2：23.2%

这说明数据集实际上以 **pre-K 到二年级** 为主，至少在本地统计中没有单独出现 grade3。论文背景中提到覆盖 pre-K 到三年级，但实际文件中的年级字段分布应进一步核对：

1. grade3 是否被合并到其他标签；
2. 是否存在大小写、命名或目录命名差异；
3. 是否 metadata 与实际数据版本不完全一致；
4. 是否部分题目缺失年级信息。

如果确实没有 grade3 样本，则不能简单地将数据集描述为均匀覆盖 pre-K 到三年级。

此外，年级分布不均衡。kindergarten 样本量是 prek 的约 2.08 倍。这种不均衡会影响：

- 按年级报告的总体准确率；
- 模型是否真正具备跨年级泛化能力；
- 技能分布与年级差异的解释；
- 训练过程中低年级视觉计数任务对模型的主导程度。

---

## 3.2 技能分布

13 类技能频次如下：

| 技能 | 数量 | 占总题数比例 |
|---|---:|---:|
| geometry | 43,493 | 40.5% |
| counting | 37,517 | 34.9% |
| comparing | 14,809 | 13.8% |
| spatial | 10,517 | 9.8% |
| scene | 9,916 | 9.2% |
| pattern | 9,151 | 8.5% |
| time | 8,875 | 8.3% |
| fraction | 7,856 | 7.3% |
| estimation | 7,627 | 7.1% |
| algebra | 7,595 | 7.1% |
| measurement | 6,412 | 6.0% |
| commonsense | 5,745 | 5.3% |
| probability | 5,247 | 4.9% |

技能总频次为 174,760，除以题目数 107,439，得到每题平均：

\[
174,760 / 107,439 \approx 1.63
\]

与给出的论文统计一致。

技能分布非常不均衡：

- geometry 和 counting 合计约占所有技能标注的 **46.3%**；
- probability 仅约 4.9%；
- commonsense、measurement、algebra、estimation、fraction 也明显低于 geometry；
- counting 和 geometry 是数据集的核心能力，而不是 13 类技能均衡覆盖。

这意味着总体准确率很可能主要反映：

1. 几何形状识别；
2. 视觉计数；
3. 简单比较；
4. 低阶空间关系。

如果模型在 probability、measurement 或 algebra 上表现很差，但在 geometry/counting 上表现很好，总体分数仍可能较高。因此仅报告 overall accuracy 会掩盖技能层面的真实能力差异。

---

## 3.3 多技能问题比例

每题技能数量为：

| 技能数 | 数量 | 占比 |
|---:|---:|---:|
| 1 | 47,808 | 44.5% |
| 2 | 51,941 | 48.4% |
| 3 | 7,690 | 7.2% |

其中约 **55.6%** 的题目包含至少两个技能，说明 IconQA 并非完全由单步题构成。

不过，多技能标签不一定等于真正的多步推理。一个题目同时标记 geometry 和 counting，可能只是“数三角形”；它在标签上包含两个技能，但推理链条仍然很短。因此在分析模型推理能力时，需要区分：

- 多标签；
- 多步骤；
- 多实体关系；
- 多轮中间状态更新；
- 需要显式算术操作的问题。

fill_in_blank 平均每题 1.81 个技能，高于整体平均值 1.63，说明填空题可能更偏向综合性或开放式推理。这个现象使 fill_in_blank 具有更高评测价值，但其样本量只占 16.9%。

---

## 3.4 问题长度

问题平均长度为 **8.3 词**，中位数为 **6 词**，最长为 **73 词**。

这表明大多数问题语言形式简短、句法简单，符合小学教材和低年级任务特点。其优势是：

- 语言理解负担低；
- 更容易聚焦视觉理解；
- 适合研究视觉推理本身；
- 有利于控制变量。

但这也带来一个限制：IconQA 对复杂语言理解、长距离指代、复杂语篇推理的覆盖有限。最长 73 词的样本可能是少数异常或复杂题目，不代表整体语言难度。

---

# 四、答案分布与统计偏差

## 4.1 小整数答案高度集中

fill_in_blank 和 choose_txt 合并后的答案文本 top10 为：

| 答案 | 数量 |
|---|---:|
| 1 | 12,563 |
| 0 | 11,612 |
| 2 | 7,185 |
| 3 | 5,112 |
| 4 | 2,883 |
| 5 | 1,654 |
| 6 | 1,305 |
| 10 | 516 |
| 8 | 478 |
| 7 | 449 |

仅答案 `"1"` 和 `"0"` 就有 24,175 次，占合并答案中的较大比例；前几个小整数占据绝对主导地位。

这会产生至少三类问题。

### 第一，答案先验非常强

如果某一类题目经常要求计数，模型可能学习到：

- 小数字比大数字更常见；
- 0、1、2、3 是高概率输出；
- 某些技能对应固定范围的答案。

在填空任务中，模型即使没有正确理解图像，也可能通过答案先验得到明显高于均匀随机的结果。

### 第二，语言模型先验可能压过视觉证据

对于 MLLM 而言，输出 `"1"`、`"2"`、`"3"` 的语言概率通常远高于不常见答案。模型可能先生成最常见的小整数，再弱使用图像信息。

### 第三，模型分数与真正视觉能力脱钩

如果答案空间高度集中，则 exact match 结果可能受到标签分布影响，而不是完全反映图像理解能力。

建议至少报告：

- 常数答案基线；
- 按答案频率的 majority-class baseline；
- 随机基线；
- 去除答案先验后的 balanced accuracy；
- 按答案值分组的准确率。

---

## 4.2 选择题的盲猜基线

选择项数量统计为：

| 选项数 | 数量 | 占总题数 |
|---:|---:|---:|
| 0 | 18,189 | 16.9% |
| 2 | 47,373 | 44.1% |
| 3 | 22,865 | 21.3% |
| 4 | 10,511 | 9.8% |
| 5 | 8,501 | 7.9% |

在非填空题中，选择题总数为：

\[
107,439 - 18,189 = 89,250
\]

如果按选项数进行均匀随机猜测，期望准确率为：

\[
\frac{
47,373/2+
22,865/3+
10,511/4+
8,501/5
}{89,250}
\approx 39.9\%
\]

因此，整个选择题部分的加权随机基线约为 **39.9%**。

这非常重要，因为单一的“选择题准确率”容易误导：

- 2 选项题随机基线是 50%；
- 3 选项题随机基线是 33.3%；
- 4 选项题随机基线是 25%；
- 5 选项题随机基线是 20%。

由于 2 选项题占选择题的 **53.1%**，模型只要在二选一问题上表现不错，就可以显著抬高总体得分。

因此应报告：

1. 2、3、4、5 选项分别的准确率；
2. 相对于随机基线的提升；
3. 宏平均准确率，而不是只报告按样本加权的总准确率；
4. 按任务类型分别报告 choose_img 和 choose_txt。

---

## 4.3 答案位置分布

选择题答案位置计数为：

| 答案位置 | 数量 |
|---|---:|
| idx0 | 35,481 |
| idx1 | 35,721 |
| idx2 | 11,981 |
| idx3 | 4,390 |
| idx4 | 1,677 |

idx0 和 idx1 基本平衡，说明随机打乱选项在整体上较好地避免了明显的位置偏差。

但 idx2、idx3、idx4 的数量逐步下降，主要原因不是答案被人为放置在前面，而是：

- 二选题最多；
- 三选题次之；
- 四选题和五选题更少。

因此，不能仅凭全局位置分布判断存在答案位置偏差。应进一步计算：

\[
P(\text{answer position}=i \mid \text{number of choices}=k)
\]

如果在每个选项数条件下各位置接近均匀，则位置偏差较小；如果某些题型或技能在特定位置上明显偏斜，则仍可能存在 shortcut。

还需要关注：

- 同一模板是否总是把正确答案放在固定位置；
- 选项随机化是否只发生在元数据层，图像内容和文本内容是否同步更新；
- 训练集与测试集是否出现相同问题但不同选项顺序。

# 五、数据质量与潜在 Shortcut 风险

## 5.1 题型不均衡

choose_img 占 53.7%，是 fill_in_blank 的约 3.17 倍。

这会导致训练模型时：

- 模型更倾向学习分类式选择策略；
- 开放式生成能力受到弱化；
- 训练损失主要由 choose_img 主导；
- 总体性能更接近视觉选项匹配能力，而不是开放问答能力。

如果将三个子任务混合训练而不加权，模型可能在 fill_in_blank 上表现较差，尤其是：

- 数字生成；
- 答案格式规范化；
- 细粒度计数；
- 需要独立计算而非选项排除的问题。

---

## 5.2 视觉模板和数据生成模式风险

IconQA 来源于小学数学教材，并经过众包收集和筛选。这种来源具有较强的模板化特征。潜在风险包括：

- 相同或近似的图标布局出现在训练和测试中；
- 问题句式与图像模板高度对应；
- 同一视觉场景只改变问题文本；
- 同一问题模板只改变数字或颜色；
- 选项图与主图之间存在固定的排版或风格差异；
- 某些技能通过背景、边框、图像尺寸或文件命名间接泄漏。

如果数据划分主要按问题实例进行，而不是按图像模板、场景、对象组合或题目来源进行，那么测试集可能包含训练集近邻样本。此时模型可以依靠模板记忆，而不是学习可迁移的视觉推理规则。

建议检查：

1. 图像感知哈希重复率；
2. OCR 或问题文本近重复率；
3. 图标组合和布局的重复率；
4. 训练/验证/测试中的场景模板重叠；
5. 相同 `pid`、相似 `label`、相同问题模板跨 split 情况；
6. Icon645 与 IconQA 是否存在图像级或图标实例级重合。

---

## 5.3 图像风格与自然图像预训练偏差

IconQA 的图像是抽象图标，而非自然照片。图像通常具有：

- 规则轮廓；
- 纯色或低纹理背景；
- 简化几何形状；
- 明确但非真实的物体关系；
- 颜色、数量和位置是关键线索；
- 物体之间可能没有真实世界尺度。

这会使自然图像预训练 backbone 的优势不能直接迁移。论文中自然图像预训练模型表现不佳，具有合理性：

- 自然图像模型擅长纹理、材质和自然物体外观；
- IconQA 更强调精确计数、边界识别、图形拓扑和组合关系；
- 抽象图标中的一个像素级形状差异可能决定答案；
- 真实世界常识在图标场景中有时不成立。

但需要注意，这种困难并不等同于高层次智能困难。它更多体现为一种不同于自然照片的感知分布偏移。

---

## 5.4 选项相关 shortcut

在 choose_img 中，模型可能利用非语义线索，例如：

- 候选图像的尺寸；
- 边缘空白；
- 图像压缩质量；
- 背景颜色；
- 候选图的复杂度；
- 选项图与问题图的视觉风格相似度；
- 正确图标与主图之间的像素统计相似性。

在 choose_txt 中，可能存在：

- 选项词频偏差；
- 选项长度偏差；
- 选项语义极性偏差；
- 某些技能与特定词语强共现；
- 数字答案与问题词法结构之间的关联。

例如，“How many” 类问题可能在训练集大量对应 1、2、3 等小整数，模型可以在视觉证据不足时产生高概率答案。

---

## 5.5 颜色和计数问题的特殊风险

许多题目依赖颜色区分，例如：

> How many shapes are purple?

这对视觉模型而言看似简单，但仍可能受到：

- 图标颜色相近；
- 抗锯齿导致的边界色；
- 图片压缩；
- 背景与前景色相似；
- 不同图标层次遮挡；
- 颜色命名不统一。

建议在评估中增加颜色鲁棒性测试，例如：

- 轻微亮度变化；
- 色调变化；
- 灰度化；
- 背景替换；
- 颜色名称替换；
- 对相同布局进行颜色置换。

如果模型准确率随颜色变化显著下降，则说明其学习的是颜色模板，而不是更一般的对象和关系概念。

---

# 六、对现代 VLM/MLLM 的适用性

## 6.1 难度定位：低语言难度，中等视觉推理难度

IconQA 的课程内容属于 pre-K 至小学低年级数学，因此从知识难度看不高，主要涉及：

- 数数；
- 颜色和形状识别；
- 大小比较；
- 简单空间关系；
- 基础分数；
- 时间；
- 简单概率；
- 基础模式和测量。

对于当前具备强视觉编码器和语言能力的大型多模态模型，许多单步题目可能偏简单，尤其是：

- 图像中物体数量少；
- 选项只有两个；
- 问题只有 5～8 个词；
- 答案是 0～5 的小整数；
- 图形差异明显。

但不能因此认为 IconQA 完全没有挑战。其挑战主要来自：

1. **抽象图形而非自然物体**；
2. **多个相似对象的精确计数**；
3. **图形和位置关系的组合**；
4. **视觉答案选项之间的细粒度差异**；
5. **需要将视觉关系映射为小学数学概念**；
6. **多技能题中的组合推理**；
7. **模型不能总是依赖真实世界常识**。

因此，IconQA 的合理难度定位是：

> 对人类小学生来说是基础数学任务；对现代 MLLM 来说，语言和知识难度较低，但抽象视觉感知、计数精度和结构化推理仍有研究价值。

---

## 6.2 作为预训练数据的价值

IconQA 可用于多模态预训练或继续预训练，尤其适合以下能力：

### 1. 视觉计数和离散属性建模

模型可以学习：

- 对象数量；
- 颜色；
- 形状；
- 尺寸；
- 位置；
- 简单集合关系。

### 2. 抽象视觉场景理解

与自然图像相比，图标场景中的信息更结构化。模型需要建立类似“对象—属性—关系”的内部表示，这对于视觉程序归纳和结构化推理有帮助。

### 3. 视觉问答格式学习

三个子任务覆盖了：

- 分类式答案；
- 文本选择；
- 开放式生成。

因此可用于构造多任务训练目标。

### 4. 低成本验证视觉推理模块

由于场景简洁、答案明确，适合作为新视觉编码器、新型跨模态融合模块或结构化推理模块的快速验证数据集。

但作为大规模通用预训练数据，IconQA 仍有限制：

- 语言多样性不足；
- 场景真实感不足；
- 答案空间狭窄；
- 问题主要围绕小学数学；
- 图像模板和风格较集中；
- 视觉概念与自然世界之间存在域差异。

因此它更适合做 **领域型预训练、视觉数学专项预训练或多模态能力补充数据**，不适合作为通用视觉语言预训练的主要数据源。

---

## 6.3 作为 SFT 数据的价值

IconQA 很适合用于监督微调，尤其是：

- 训练视觉问答格式；
- 训练短答案生成；
- 训练图像选项选择；
- 训练视觉属性和关系推理；
- 训练模型输出规范化的数字或短文本。

建议将不同子任务采用不同提示格式。例如：

```text
Question: How many shapes are purple?
Answer with only the number.
```

对于 choose_img：

```text
Select the correct image option. Output only the option index.
```

对于 choose_txt：

```text
Choose one answer from the listed options. Output the exact option text.
```

不过，SFT 时不能只优化总体准确率。需要避免模型过度学习：

- 小整数答案先验；
- 固定问题模板；
- 选项位置；
- 特定技能标签与答案词的共现；
- 图片版式。

建议对训练样本进行任务和技能加权，避免 choose_img 占比过高。

---

## 6.4 作为评测集的价值和局限

### 价值

IconQA 可用于评测：

- 视觉计数；
- 抽象形状识别；
- 视觉比较；
- 空间关系；
- 视觉概率；
- 图像选项匹配；
- 多技能组合；
- 从视觉场景生成数字答案。

它特别适合做：

- 小规模模型能力探针；
- 视觉编码器对比；
- 抽象图形理解测试；
- 多模态推理模块消融实验；
- 视觉 grounding 和符号推理实验。

### 局限

IconQA 不适合单独作为通用 MLLM 的综合评测，原因包括：

1. 数学范围主要限于小学低年级；
2. 问题文本较短；
3. 答案主要是小整数或固定选项；
4. 图像风格和分布较集中；
5. 很多题目可能可通过简单计数或模式识别解决；
6. 存在模板记忆和数据近重复风险；
7. 选择题随机基线并不统一；
8. 题型和技能分布不平衡。

因此，模型在 IconQA 上取得高分，不能直接推导出其具备广泛的空间推理、数学推理或通用视觉理解能力。

---

# 七、实际使用建议

## 7.1 推荐的数据使用划分

### 方案 A：分任务训练与评测

建议分别训练或至少分别统计：

1. `choose_img`
2. `choose_txt`
3. `fill_in_blank`

不要只将三类任务混合后报告单一 accuracy。

推荐报告：

| 任务 | 必报指标 |
|---|---|
| choose_img | accuracy、按选项数分组 accuracy |
| choose_txt | accuracy、按选项数分组 accuracy、选项文本 exact match |
| fill_in_blank | exact match、数值归一化 accuracy、按答案大小分组 accuracy |

---

### 方案 B：多任务训练、任务均衡采样

如果目标是训练 MLLM，建议使用任务均衡采样，例如：

- 每个任务按相近概率采样；
- 或按 `1 / sqrt(任务频次)` 进行温度采样；
- 或为 fill_in_blank 设置更高损失权重。

一个简单方案是每个 batch 中固定包含：

- 1/3 choose_img；
- 1/3 choose_txt；
- 1/3 fill_in_blank。

这样可以避免 choose_img 的 53.7% 样本占比进一步放大。

---

## 7.2 是否需要清洗

建议进行以下清洗。

### 1. 元数据一致性检查

至少修正或显式记录：

- val 少 1 条；
- metadata 与实际年级标签是否一致；
- `answer` 类型是否统一；
- 选择题答案索引是否从 0 开始；
- 选项顺序与 answer 索引是否同步；
- 图像文件是否缺失或损坏。

### 2. 图像质量检查

检查：

- 主图是否能正常打开；
- choice 图是否缺失；
- 图像尺寸是否异常；
- 透明背景是否被错误处理；
- 颜色空间是否统一；
- 是否有空白图、重复图或近重复图。

### 3. 文本规范化

对于填空答案，建议同时保留：

- 原始答案字符串；
- 数值解析结果；
- 规范化答案。

例如处理：

- `"1"` 与 `"1 "`；
- `"two"` 与 `"2"`；
- 大小写差异；
- 标点；
- 复数形式；
- 小数、分数或时间格式。

评测时应同时报告严格 exact match 和语义/数值归一化匹配，避免不同格式导致不合理扣分。

---

## 7.3 是否需要重平衡

不建议简单删除大量 geometry 或 counting 样本，因为它们是 IconQA 的核心内容。但建议构建多个版本：

### 原始分布版本

用于：

- 与论文结果比较；
- 复现已有基准；
- 保持数据集原貌。

### 任务均衡版本

使三个子任务贡献接近，用于：

- 多任务 SFT；
- 比较不同任务的学习效率；
- 避免 choose_img 主导训练。

### 技能均衡版本

对 13 类技能进行重采样或损失加权，用于：

- 技能级能力评测；
- 减少 geometry/counting 对整体指标的支配；
- 研究长尾技能。

### 答案平衡或先验控制版本

不一定要人为修改原题，但可以：

- 对小整数答案进行分层采样；
- 在训练中对低频答案加权；
- 建立答案先验校正基线；
- 单独评估模型在高频和低频答案上的性能。

---

## 7.4 推荐的评测指标设计

### 基础指标

1. 任务级 accuracy；
2. 严格 exact match；
3. 数值归一化准确率；
4. 选择题按选项数分组准确率；
5. 每项技能准确率；
6. 每个年级准确率；
7. 每个任务 × 技能交叉准确率。

### 更稳健的汇总指标

建议同时报告：

- micro accuracy：按题目数量加权；
- macro accuracy：先按技能或任务平均，再汇总；
- balanced accuracy：对答案类别或技能进行平衡；
- relative gain over random：相对随机基线的提升；
- worst-group accuracy：最弱技能/年级组准确率。

例如，对于选择题可以报告：

\[
\text{Normalized Gain}
=
\frac{\text{Model Accuracy} - \text{Random Accuracy}}
{1-\text{Random Accuracy}}
\]

这样 2 选项和 5 选项题的结果更可比较。

### 推荐的分组分析

至少按以下维度交叉报告：

- 子任务；
- 选项数量；
- 年级；
- 13 类技能；
- 技能数量；
- 答案值；
- 问题长度；
- 图像中对象数量；
- 是否需要颜色识别；
- 是否需要空间关系；
- 是否需要多步计算。

特别建议绘制：

- 技能 × 题型热力图；
- 年级 × 技能热力图；
- 选项数 × 准确率曲线；
- 答案频率 × 准确率曲线；
- 问题长度 × 准确率曲线。

---

## 7.5 建议增加的 shortcut 检测实验

为了判断模型是否真正使用图像，建议加入以下对照实验。

### 1. 文本-only baseline

只输入问题和文本选项，不输入图像。若文本-only 模型得分较高，说明存在明显语言或答案先验。

### 2. 图像-only baseline

不输入问题，只输入图像或候选图像。若仍然有较高准确率，说明图像模板可能泄漏标签。

### 3. shuffled-image baseline

随机交换问题图像，保持问题和选项不变。理想情况下准确率应大幅下降。

### 4. shuffled-question baseline

打乱问题文本，保持图像不变。用于检测模型是否只依赖图像模式。

### 5. answer-position baseline

只输入选项位置或选项数量，测试位置偏差。

### 6. majority-answer baseline

预测全局最高频答案，或按技能、年级预测条件最高频答案。

### 7. template-held-out split

按照问题模板、图像布局或图标组合进行划分，避免近重复样本跨 split。

这些实验比单纯报告一个高 accuracy 更能说明模型是否真正完成了视觉语言推理。

# 八、Icon645 的配合使用建议

Icon645 包含：

- 645,687 张彩色图标；
- 377 类图标；
- 用于视觉骨干预训练。

它与 IconQA 的关系可以理解为：

- Icon645：偏视觉表征学习；
- IconQA：偏视觉语言任务和推理学习。

## 8.1 推荐训练流程

### 阶段一：Icon645 视觉预训练

可以训练或继续训练视觉编码器，使其学习：

- 图标形状；
- 颜色；
- 轮廓；
- 类别差异；
- 小尺度对象；
- 抽象图形结构。

### 阶段二：IconQA 多任务 SFT

使用三种问答任务进行联合训练：

- choose_img；
- choose_txt；
- fill_in_blank。

### 阶段三：IconQA 技能级评测

在不改变测试集的情况下，按：

- 技能；
- 年级；
- 题型；
- 选项数；
- 答案频率；

进行细粒度评估。

---

## 8.2 Icon645 使用时的泄漏风险

必须检查 Icon645 和 IconQA 之间的重叠关系。因为 Icon645 可能包含：

- 与 IconQA 主图相同的图标实例；
- 相同图标类别；
- 相同视觉素材；
- 与测试集近似的图标组合。

如果用 Icon645 进行视觉预训练后再在 IconQA 上评测，需要明确区分：

1. **类别级重叠**：训练中见过相同类型的图形；
2. **实例级重叠**：训练中见过测试图像中的具体图标；
3. **场景级重叠**：训练中见过相同布局或模板；
4. **像素级或近重复重叠**：训练样本与测试样本几乎相同。

类别级重叠通常是允许的，类似 ImageNet 预训练；但实例级和场景级重叠可能造成显著泄漏，尤其在测试模型是否具备组合推理时。

建议公开两套结果：

- 使用 Icon645 预训练；
- 不使用 Icon645 预训练。

这样可以衡量 Icon645 对 IconQA 的真实增益。

# 九、最终结论

IconQA 是一个具有明确研究价值的抽象图标视觉问答数据集。它的核心优势不是高阶数学，而是：

- 规模较大；
- 任务形式丰富；
- 视觉输入结构清晰；
- 覆盖多类小学数学技能；
- 能测试模型对抽象图形、数量和关系的理解；
- 能揭示自然图像预训练模型在图标域上的迁移问题。

从本地实际统计看，数据集存在几个必须正视的结构性特征：

1. **choose_img 占 53.7%**，显著主导整体分布；
2. **约 44.1% 的全部题目是二选一**，选择题总体随机基线约为 39.9%；
3. **答案高度集中于 0、1、2、3 等小整数**，存在明显答案先验；
4. **geometry 和 counting 占绝大多数技能频次**，长尾技能容易被总体指标掩盖；
5. **约 55.6% 题目包含两个或更多技能标签**，但多标签不一定等于真正多步推理；
6. **问题平均只有 8.3 词**，语言难度较低；
7. **val 少 1 条**属于小型数据管理瑕疵，应在复现实验中记录；
8. **模板重叠、近重复图像、答案位置和选项统计特征**可能造成 shortcut；
9. **Icon645 能有效帮助图标视觉预训练，但必须检查跨数据集泄漏**。

因此，最合理的使用方式是：

- 不把 IconQA 当作单一总体 accuracy 基准；
- 按三类任务分别训练和评估；
- 按选项数、技能、年级和答案频率分组；
- 同时报告随机、majority、text-only、image-only 等基线；
- 对数据进行近重复和模板泄漏检查；
- 使用 Icon645 时区分类别级迁移与实例级泄漏；
- 将 IconQA 作为抽象视觉推理和小学数学视觉问答专项基准，而不是通用多模态智能的完整代表。

总体评价可以概括为：

> IconQA 是一个规模较大、任务设计清晰、对抽象图标理解具有独特价值的数据集；但其答案空间、题型结构和技能分布存在明显偏差，必须通过分组指标、平衡采样和 shortcut 检测，才能获得可信的模型能力结论。

# 十、补充分析：MuirBench Diagram Understanding 与 IconQA choose_img 的关系

> 本节为后续追加的实证分析（非 gpt-5.6-sol 生成），基于本地 MuirBench 原始数据（`/inspire/hdd/project/video-understanding/public/personal/wran/projects/muirbench`，5 个 test parquet，2,600 题）与本地 iconqa_data 的全量比对。

## 10.1 背景

[MuirBench](https://arxiv.org/abs/2406.09411)（Wang et al., 2024）是一个多图理解评测基准，含 11,264 张图、2,600 道多图选择题、12 个任务。其论文 §A.2 写明 **Diagram Understanding 任务的数据直接取自 IconQA**（"existing data come from ... IconQA"，每个来源采样至多 200 条）。本节用本地数据验证并量化了这一对应关系。

## 10.2 规模与结构

- MuirBench 中 `task = Diagram Understanding` 共 **398 题**（占全基准 15.3%），全部标注 `image_type = Graphics`、`image_relation = Cropped/Zoomed`。
- 398 题 = **199 对 pairwise 实例**（`counterpart_idx` 两两配对，无落单）：
  - 199 个 **answerable**：原样取自 IconQA choose_img；
  - 199 个 **unanswerable counterpart**：图像列表与 answerable **逐张完全相同**（199/199 对全部如此），仅对问题做最小改动（实测样例："area of **5**" → "area of **6**"、"**one less**" → "**one more**"），使所有候选图均不正确；
  - 所有 398 题的选项中都新增了 **"None of the choices provided"**，它恰是 199 个 unanswerable 题的正确答案（answerable 题从不以它为答案）。
- 答案字母分布：A 104 / B 114 / C 140 / D 21 / E 19，基本均衡。
- 每题图数：3 张 205 题、4 张 53 题、5 张 52 题（主图 1 张 + 候选图 2–4 张），与 IconQA choose_img 的选项数分布一致。

## 10.3 对应关系的实证证据

**问题文本匹配**：199 个 answerable 题中，**193 题（97.0%）** 的问题文本（去除 `<image>` 占位符后）在 IconQA choose_img 中逐字存在。其余 6 题与 IconQA 问题仅有极小措辞差异（同模板换了属性词，如 "Which heart is **brown**?" vs 本地 "Which heart is **orange**?"）。

**图像匹配**（因 MuirBench 将图转存为 JPEG，文件头 `\xff\xd8`，与本地 PNG 无任何像素级完全相同的图，故采用 16×16 感知哈希初筛 + 同尺寸候选像素级平均差验证）：

- MuirBench 的 1,404 张图中 **98.7%**（1,386 张）能在 IconQA choose_img 中找到感知哈希距离为 0 的图；
- 按"问题逐字相同 + 主图和全部选项图尺寸一致"联合约束精确匹配到 **155 题**，其中 108 题整套图的平均像素差 < 6（JPEG 重压缩水平），判定为同一实例；
- 这 108 题的 IconQA pid 归属 split：**test 82 / train 20 / val 6**；其中 16 题像素近完全一致（meandiff < 2），**全部落在 test split**。
- 整体与论文"sample up to 200 test examples"（从 test split 采样）一致；train/val 的命中主要源于 IconQA 自身跨 split 存在同模板近重复实例。

**与本地 iconqa_data（v1, 2021.06.21）的差异**：

1. MuirBench 图像为 JPEG 重编码，本地为 PNG，像素差 2–6 为压缩痕迹；
2. 部分实例与本地同问题 pid **同模板但图标排列或颜色不同**（如实测 "Which heart is brown?" 一题，主图中绿/紫/棕心的左右顺序不同）——说明 MuirBench 使用的 IconQA 快照版本与本地 zip 不完全一致，或实例经过重新渲染；
3. 格式转换：IconQA 的 `image.png` → 问题中的 `<image>` 占位符，`choice_*.png` → 图片选项，答案由索引转为字母（A–E）。

## 10.4 对数据使用的含义（污染风险）

- **评测污染**：MuirBench 的 Diagram Understanding（占基准 15.3%）与 IconQA **test split** 高度重合，且 IconQA train/val 中存在大量同模板、仅颜色/数值不同的近重复题。任何在 IconQA（尤其 train split）上训练或微调过的模型，在该子任务上的得分会显著虚高，不能视为泛化能力。
- **反方向同理**：若以 MuirBench 为评测集筛选模型，再回测 IconQA test，也会遇到实质相同的题目。
- **unanswerable 设计的局限**：Diagram Understanding 的 unanswerable 变体全部通过"改问题数字/属性词 + 新增 None 选项"构造，模式单一；模型只要学会"数字对不上就选 None of the choices provided"的浅层规则即可拿分，鲁棒性声称需打折。
- **建议**：
  1. 训练 IconQA 时如需保留干净的 MuirBench 评测，应按问题模板（label）对 train/val/test 做去重或按模板划分；
  2. 报告 MuirBench 结果时将 Diagram Understanding 单独列出，注明其与 IconQA 的同源性；
  3. 若需严格排除，可基于本节生成的 398↔pid 映射（随报告归档：`muir_iconqa_mapping/precise_map.json`、`row_pid_map.json`、`full_match.json`）对训练集做实例级过滤。

## 10.5 验证方法记录（可复现）

1. 文本：去除 `<image>` 占位符后与 `problems.json` 中 `ques_type == choose_img` 的 question 精确匹配；
2. 图像：16×16 灰度平均哈希（ahash）对 IconQA choose_img 全部 208,266 张图（57,672 题的主图 + 选项图）建库，汉明距离 ≤ 20 且尺寸相同者为候选，再计算 RGB 像素平均绝对差（meandiff），< 2 视为同图重压缩、2–6 视为 JPEG 级重编码、> 6 视为不同实例；
3. 配对结构：按 `counterpart_idx` 配对，比较图像列表的像素哈希集合与选项中 "None of the choices provided" 的位置。

# 十一、补充分析：基于 Icon645 素材库构造多图 QA 数据的方案

> 本节为后续追加（非 gpt-5.6-sol 生成），基于本地 `../icon645`（即 Icon645：377 类、645,687 张彩色 PNG，多数 512×512 RGB，按类分目录）的核实结果，结合 MuirBench 使用 IconQA 的模式，总结用 icon 素材库构造多图 QA 数据的方案。

## 11.1 核心模式：程序化"画布渲染 + 模板出题"

icon 素材的最大优势是干净、无背景、类别明确——可以把 N 个 icon 以受控的类别/颜色/数量/位置/大小摆到空白画布上，**渲染过程本身就是 ground truth**，无需人工标注。这正是 IconQA（题库来自 IXL 教材，标注成本高）和 MuirBench（人工改造，仅 398 题）都不具备的可扩展性。

## 11.2 可构造的任务类型（IconQA 13 技能 × MuirBench 多图关系）

**单图出题（作为多图基元）**：计数、颜色/属性识别、空间关系（上下左右/旁边）、比较（最多/最少）、奇偶分类、odd-one-out。

**多图组合（借鉴 MuirBench 的 10 种 image relation）**：

| 构造类型 | 对应 MuirBench relation / 任务 | 做法 |
|---|---|---|
| 图选项 MCQA | Diagram Understanding | 一张场景主图 + 2–5 张候选图作选项（如"哪张图的苹果比主图多一个？"），干扰项可控：数量±1、同色不同类、同类不同色、镜像布局 |
| 相似性判断 | Overall/Partial Similarity | 多图共享部分 icon，问"哪两张图有相同的物体组合" |
| 找不同 | Difference Spotting | 同场景渲染两次、改 1–2 处（换色/增删/移位），问差异数量或位置 |
| 局部匹配 | Cropped/Zoomed | 大画布裁局部，问"哪张小图是大图的一部分"（MuirBench 给 diagram 标的正是此 relation） |
| 序列/排序 | Temporal / Ordering | 渲染序列帧（数量递增、位置移动、颜色循环），问下一帧或排序——icon 数据模拟 temporal relation 的独特价值，可作为静态多图到视频理解的过渡 |
| 跨图计数/比较 | Counting | "三张图里一共有多少气球？""哪张图的红色物体最多？" |
| 视觉检索 | Visual Retrieval | 同一 icon 实例混入不同场景，"哪张图含有与参考图完全相同的物体" |

## 11.3 关键构造机制（MuirBench 经验的工程化）

- **Pairwise 设计**：每个可答题配一个不可答 counterpart，三种策略均可程序化——改问题参数（数量±1）、替换/重排图像、替换选项，并把 "None of the choices provided" 加入选项。成本比 MuirBench 的人工改造低几个量级。
- **干扰项分级难度**：easy（不同类）→ medium（同类不同色/数）→ hard（同组合差 1 个/镜像），按难度分层采样。
- **答案分布强制均衡**：针对 IconQA 暴露的问题（答案扎堆小整数、二选一占 44%、选项位置不均），生成时按目标分布采样：答案值均匀、选项数 2–5 配比、正确答案位置均匀、控制 None 选项占比。
- **技能/元数据自动标注**：类别、数量、空间关系、所需技能（可沿用 IconQA 13 技能体系）在渲染时全部已知，评测可直接做技能×题型×难度交叉分析，避免 IconQA 技能长尾被 overall accuracy 掩盖的问题。
- **防泄漏划分**：按"场景模板 + icon 组合"划分 train/test，而非按实例随机分；渲染图做感知哈希去重。
- **问题文本多样化**：模板生成骨架后用 LLM 做 paraphrase 和语言难度分级，避免纯模板句式带来的 shortcut。

## 11.4 注意事项

- **类间素材量不均**：arrow 类约 20k 张，部分类不足千张，采样需按类配额，否则分布被大类主导。
- **与 IconQA/MuirBench 的泄漏**：Icon645 与 IconQA 同源（Flaticon + IXL），若仍需在 IconQA 或 MuirBench 上评测，应按 icon 实例哈希排除重叠（参照第 10.4 节的污染分析和 `muir_iconqa_mapping/` 映射）。
- **能力定位**：377 类扁平风格 icon 适合训练"结构化视觉推理"（计数、关系、组合、序列），不能替代自然图像数据。合理定位是**受控的多图/帧序列推理训练与诊断数据**，正好补上 IconQA 静态单题与真实视频之间的空档。

## 11.5 一句话总结

> IconQA 提供题型与技能蓝图，MuirBench 提供多图关系与 pairwise 健壮性设计，Icon645 提供可编程素材——三者组合可以低成本合成带精确 ground truth、分布可控、难度可调的多图（乃至帧序列）QA 数据。
